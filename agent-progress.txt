[database] Build database access utilities (lib/db.ts)  — passes=true
  - Created lib/db.ts: shared SQLite handle + typed helpers.
  - Singleton getDb() (lazy open, cached) with WAL + foreign_keys pragmas.
  - query<T>(sql, params) -> rows; get<T>(sql, params) -> row|null;
    run(sql, params) -> { changes, lastInsertRowid } (normalized to numbers).
  - Prepared-statement cache; all SQL parameterized via $name binds.
  - closeDb() for test isolation.
  - DESIGN DECISION (spec-sanctioned, SPEC §3.2/§17): thin driver abstraction.
    Uses bun:sqlite under Bun (dev/build/start run with --bun) and falls back to
    node:sqlite (DatabaseSync) under plain Node, so the data layer is testable
    under Vitest (which runs on Node) without breaking production. Driver picked
    at runtime via createRequire + a variable module specifier (bundler-safe).
  - Verified: dual-runtime smoke test (Bun + Node, all asserts pass), tsc via
    `bun run build` (OK), `bun run lint` (clean).

[auth] Configure better-auth server (lib/auth.ts)  — passes=true
  - Created lib/auth.ts: single betterAuth() instance, the authoritative server
    auth surface. emailAndPassword enabled; nextCookies() plugin last (forwards
    Set-Cookie from server-side calls/actions). secret + baseURL auto-read from
    BETTER_AUTH_SECRET / BETTER_AUTH_URL env.
  - DB: passes the SHARED handle from getDb() (SPEC §6 "one file, two consumers").
    Cast to bun:sqlite Database type (driver-agnostic getDb() returns the real
    bun:sqlite instance under the Bun runtime, which better-auth supports natively).
  - Migrations: better-auth never auto-migrates, so on cold start we run them
    programmatically — getMigrations(auth.options) -> runMigrations() (idempotent),
    imported from "better-auth/db/migration". Guarded by NEXT_PHASE !==
    "phase-production-build" so `next build` does no DB work.
  - Verified end-to-end via a throwaway Bun smoke test (since vitest/Node can't
    drive bun:sqlite-backed auth): migrations create all 4 tables (user, session,
    account, verification); getSession(no cookie) -> null; signUpEmail persists a
    user row + an account row (providerId "credential", hash in account not user).
    Smoke file + its DB removed after. `bun run build` PASS (auth.ts typechecks:
    cast + top-level await OK), `bun run lint` PASS.
  - NOTE for next iteration: this ALSO created the better-auth schema, so the prd
    task "Create SQLite database with better-auth schema" is now trivially
    satisfiable — just verify the 4 tables exist and mark it passes=true.
